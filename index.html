bmlm: Multilevel Mediation Modeling with Stan

Matti Vuorre

2016-06-26

Introduction

bmlm is an R package that allows easy estimation of multilevel mediation models. It relies heavily on the RStan interface to the powerful Stan Bayesian inference engine. Users can estimate, summarize and plot a multilevel mediation model easily with the convenient functions provided with bmlm. This document explains how to install bmlm and its required components, and then walks through an example of how to use it in practice.

Installing bmlm

Pre-requisites

  1. Please ensure you have the latest version of R installed.
  2. This package depends on Stan. Please see here for how to install the RStan interface.
  3. Currently, bmlm is only available on GitHub, and therefore requires devtools for installation. If you don’t have the devtools package installed in R, install it by running the following code in the R console:
install.packages("devtools")

Installing bmlm

Then proceed to install bmlm:

devtools::install_github("mvuorre/bmlm")

If something goes wrong during the installation process, you will receive a notice usually asking you to install additional packages. If you are unable to resolve the problems, please contact me.

Example

After installing the required software, load the bmlm package to your current R workspace:

library(bmlm)

bmlm contains an example data set from Intensive Longitudinal Methods: An Introduction to Diary and Experience Sampling Research (Bolger & Laurenceau, 2013). We’ll use this data set in the example, and first load it into the workspace from the package, and display what the data looks like:

data(BLch9)

Fit model

To estimate the multilevel mediation model, simply run the following command:

fit <- mlm(d = BLch9, iter = 1000)

The main arguments to mlm() are d (a data.frame), which here was set to BLch9. The user also needs to specify which columns contain the variables needed for the mediation model, unless they are already named id, x, m, and y:

  • id is a column of participant IDs in the provided data.frame.
  • x is the manipulated variable.
  • m is the mediator variable.
  • y is the outcome variable.

We didn’t need to specify these in this example, because the defaults point to those names, and the example data already contained columns with these names.

There are various additional arguments to the above command. Most notably, the iter = 1000 specified that Stan should draw 1000 samples from the posterior distribution, for each MCMC chain. The default is to use 4 chains. Further, Stan’s MCMC algorithms use a portion of the samples as warmup to adjust various underlying parameters. The default of one half was used for this example. So we have a total of \(1000 / 2 * 4 = 2000\) samples from the joint posterior distribution. In practice, for this model we recommend obtaining more samples to ensure precision in parameter estimates. For this example, 2000 samples is sufficient.

Summarize fitted model

After the samples have been obtained, bmlm’s helper functions can be used to obtain summaries of the results. For more options, all rstan methods are also available.

Numerical summary

A numerical summary can be obtained by mlm_summary(fit):

mlm_summary(fit)
Parameter Mean SD CI_4.5 CI_95.5 pprob n_eff Rhat
a 0.19 0.04 0.13 0.25 1.0000 816.06 1
b 0.15 0.03 0.10 0.20 1.0000 755.65 1
cp 0.10 0.02 0.07 0.14 1.0000 2000.00 1
corrab 0.45 0.15 0.19 0.68 0.9985 502.65 1
ab 0.05 0.01 0.03 0.08 1.0000 874.14 1
c 0.16 0.03 0.12 0.21 1.0000 1264.28 1
pme 0.35 0.08 0.23 0.48 1.0000 1144.50 1

mlm_summary() returns, for each parameter, the following information:

  • Posterior Mean: This can be used as a point estimate of parameter
  • Posterior SD: The standard deviation of the marginal posterior distribution of plausible parameter values.
  • Lower and upper limits to Credible Intervals. The CIs summarize the central X% mass of the marginal posterior distribution, where X% is defined by a level argument to mlm_summary(). The default “confidence level” is 0.91, but users may supply any value they desire.
  • Posterior probability (pprob), which summarizes the probability density at the observed side of 0. For example, if the estimate (Mean) is -0.55, the posterior probability is defined as the proportion of probability density below zero.
  • n_eff and Rhat are diagnostic values used to diagnose the performance of the underlying Stan MCMC procedures. see ?mlm or ?stan for details.

Graphical summaries

Currently, bmlm provides two graphical summaries of the estimated model. The first draws a path diagram of the mediation model, with point estimates of the relevant average-level parameters, and their associated posterior probabilities:

mlm_path_plot(fit)

Figure 1. Path diagram of the average level mediation model, with numerical summaries of the relevant parameters.

This figure offers a quick view of what the model suggests about the data: All paths (a, b, c') are positive with very high posterior probabilities (p = 1). The upper left corner gives three more parameters: ab is the average mediated effect, c is the total effect, and %me is the percent mediated effect. The width of each arrow represents the magnitude of each relationship.

For a more detailed investigation of the model’s parameters, bmlm offers two methods for plotting the samples from each parameter’s posterior distribution. These are obtained by a call to mlm_pars_plot(type = X) where X can be either hist for histograms, or coefplot for a coefficient plot. If plotting a coefficient plot, the user may specify level to set the “confidence level”, which is represented by the length of the lines surrounding each point estimate.

mlm_pars_plot(fit, type = "coefplot", level = .50)
mlm_pars_plot(fit, type = "coefplot", level = .99)

The user can also specify the parameters to show on the plot, as illustrated in the next histograms:

mlm_pars_plot(fit, type = "hist", pars = c("ab", "c", "pme", "corrab"))

Figure 3. Histograms for a selection of parameters.

The histograms are useful for visually assessing the shape of each marginal posterior distribution.

Further information

The functions contained in bmlm have various options available for the user. Inspect these options by looking at the functions’ help pages:

?mlm
?mlm_summary
?mlm_path_plot
?mlm_pars_plot

Users can also input any valid stan() arguments to mlm(); see ?stan for details. The fitted object from mlm() is a valid Stanfit object, so all Stanfit methods are available as well. For example, it is important to assess the performance of Stan’s MCMC algorithms after estimating a model by, for example, inspecting traceplots for convergence problems:

pars <- c("a", "ab", "cp", "c", "corrab")
traceplot(fit, pars = pars)
